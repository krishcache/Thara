This document captures recurring business knowledge.

It is intended to avoid rediscovering operational knowledge repeatedly.

---

# General Principles

Technology exists to improve business outcomes.

Business value always comes before technical elegance.

---

# Automation Principles

Automate only when

- Business value exists
- Manual effort is repetitive
- Process is stable
- Maintenance effort is acceptable

Avoid automation simply because it is technically possible.

---

# Documentation Principles

Document

- Business rules
- APIs
- Dashboards
- Architecture
- Decisions

Documentation reduces dependency on individuals.

---

# Warehouse

Objectives

- Accuracy
- Visibility
- Traceability
- Standardisation

Preferred

- SharePoint
- Power BI
- Microsoft ecosystem

---

# SharePoint

Use SharePoint as the operational document platform.

Preferred structure

Department

↓

Function

↓

Process

↓

Supporting Documents

Avoid deep folder nesting.

Use permissions where required.

---

# APIs

Preferred integration order

1. Existing enterprise APIs

2. Existing file feeds

3. SharePoint

4. Manual download

Avoid introducing third-party tools unless justified.

---

# Business Improvement Philosophy

Always ask

Can this process be

- Removed?
- Simplified?
- Automated?
- Standardised?
- Reused?

Never automate poor processes.

---

# ROI Evaluation

Before implementing,

consider

- Cost
- Time
- Maintenance
- Scalability
- Business benefit

If effort exceeds benefit,

recommend parking the idea.

---

# Decision Making

Business decisions should balance

- Cost
- Risk
- Time
- User adoption
- Long-term maintenance

Never optimise for only one dimension.
